# Coinage — Visual Direction Canonical Spec

## 1. Objective
Lock Coinage visual direction before implementation so rendering and UX decisions remain consistent across all views.

## 2. Core Artistic Statement
Coinage combines:
- **Grepolis structural readability** (map-first spatial strategy, island-based reading)
- **Dark premium techno military-command-center DA** ("Ultron" shorthand)

Structural logic is inherited; artistic language is reinterpreted.

## 3. What Coinage Is Not
Coinage must not become:
- A web dashboard
- A protocol/node graph
- An analytics UI
- A noisy sci-fi cliché collage
- A flat placeholder interface
- A random asset soup with no visual system

## 4. Global Tone
Target tone across views:
- Dark
- Premium
- Strategic
- Techno
- Controlled glow (not neon chaos)
- Tension and latent conflict
- Sense of scale and geopolitical distance

## 5. Faction-Islands Language
Faction-islands must read as:
- Organic, irregular masses
- Island-like or floating district-like forms
- Varied in size and silhouette family
- Readable from distance
- Minimal in texture but still game-like and intentional

They must not read as circles, blobs, cards, UI widgets, or placeholder shapes.

## 6. Digital Ocean Language
The digital ocean is the structural equivalent of Grepolis sea.

Rules:
- Deep dark background layer
- Subtle digital life and motion
- Spatial void as meaningful gameplay structure
- Supports island readability; never overpowers it

## 7. City Slot Visual Language
City slots in Map and Faction views must be:
- Clearly visible
- Immediately readable
- Integrated into territory (not floating badge UI)
- Distinguishable for free vs occupied state
- Consistent with premium game aesthetic

They must not look like HTML labels, spreadsheet markers, or cheap icon overlays.

## 8. Detail Density Rules
- Keep detail minimal but alive
- Prioritize strategic readability over ornamental complexity
- Avoid clutter that hides silhouettes or slots
- Avoid dead-flat emptiness that kills world presence

Rule of balance: enough information to feel authored, not so much that spatial reading collapses.

## 9. Visual Priorities at Launch
Because all factions start neutral, first visual milestone must prioritize:
- Faction-island silhouette quality
- World placement/composition
- Slot readability (presence + free/occupied structure readiness)
- Digital ocean composition

Do not prioritize ownership-state FX for the first canonical visual pass.

## 10. Non-Negotiable Truths
- The map is the game surface, not a widget.
- Spatial reading comes before panel reading.
- Faction-islands must be organic and varied.
- City slots must be visible directly on factions in Map View.
- Faction View must stay spatial, never sheet-like.
- City View may be denser but must remain game-like.
- Avoid dashboard, graph, and generic SaaS aesthetics.
