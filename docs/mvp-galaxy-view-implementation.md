# MVP Galaxy View Implementation

## Background strategy

The MVP galaxy uses a lightweight deterministic starfield:
- `THREE.Points` with seeded star positions and subtle brightness variation
- dark space background color + light fog for depth layering
- slow, minimal starfield drift to avoid a static flat backdrop

This keeps the scene readable while avoiding heavy shader pipelines.

## Planet placement strategy

Placement is generated by `generateGalaxyLayout(worldSeed, config)` with deterministic seeded RNG.

Rules used:
- radial distribution inside a bounded field radius
- rejection sampling enforcing a minimum 2D spacing
- deterministic z-depth band for subtle parallax/depth cues
- stable planet ids/seeds (`planet-0`, `planet-1`, ...)

Result:
- populated but readable galaxy
- no accidental clumping/overlap below spacing threshold
- same `worldSeed` always yields same map layout

## Movement approach

Movement is implemented as map-style camera panning:
- fixed camera z (no zoom controls)
- keyboard movement (WASD / arrows)
- pointer drag panning
- camera x/y clamped to field bounds

This preserves galaxy-map framing and avoids turning the view into an orbit/cinematic viewer.

## Performance decisions

To scale from single-planet to many planets:
- profiles and layout are generated once (`useMemo`), not per frame
- shared precomputed geometry variants are reused across planets
- no per-frame planet deformation/regeneration
- no orbit controls or complex post-processing
- atmosphere is optional and lightweight

This deliberately trades some single-planet fidelity for multi-planet scene stability.

## Simplifications for MVP

Compared with an advanced renderer stack, this implementation intentionally simplifies:
- fixed-size geometry variant bank instead of unique high-detail geometry per planet
- one clean material path (`MeshStandardMaterial`) without advanced custom shader graph
- no complex galaxy overlay UI
- no dynamic LOD system in this slice

## Deferred intentionally

- Gameplay integration (join/create/select flows)
- Token/pending-planet state rendering logic
- zoom levels and strategic minimap overlays
- batching/instancing refinements and LOD tuning for very high planet counts
- advanced atmospheric/nebula effects
