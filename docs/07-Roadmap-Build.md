**COINAGE**

**Roadmap & Plan de Build**

*MVP → V0 → V1 → V2 → V3*

Version 2.0 — Avril 2026

# Cap produit

Livrer progressivement un jeu de stratégie persistant sur world map 2D sectorisée, sans changer les règles cœur.

# Phases

## MVP — Validation d’attractivité

Objectif : vérifier l’intérêt pour la boucle ville + carte sectorisée.

Inclus :

- onboarding léger
- création de ville
- production continue
- visualisation monde 2D
- attribution d’un premier secteur neutre

Exclus :

- guerres officielles
- gouvernance complète
- end game de saison

## V0 — Boucle quotidienne

Objectif : rétention jour 1 / jour 7.

Ajouts :

- unités de base
- raids
- sièges
- espionnage initial

## V1 — Connexion token et faction

Objectif : brancher la couche on-chain et la progression collective.

Ajouts :

- auth wallet
- validation holding
- Signal de faction
- branches collectives

## V2 — Politique et conflits majeurs

Objectif : structurer le jeu social et stratégique.

Ajouts :

- Conseil
- alliances
- diplomatie avancée
- guerres officielles complètes

## V3 — Saisons et monétisation mature

Objectif : fermer la boucle long terme.

Ajouts :

- end game saisonnier
- classement final
- événements spéciaux
- monétisation complète non pay-to-win

# Priorités techniques

1. Stabilité du modèle ressources
2. Fiabilité des captures territoriales
3. Lisibilité de la carte et des clusters
4. Robustesse des snapshots on-chain
5. Outillage anti-abus (combat, gouvernance, économie)
