# Audit runtime — génération procédurale des planètes (Galaxy View)

## Scope
- Source of truth: runtime code only (`src/domain/world`, `src/rendering/planet`, `src/ui/galaxy`).
- Runtime evidence: `analysis/runtime-planet-audit.json` generated by `analysis/runtime-planet-audit.ts` on 100 planètes (`planet-0` à `planet-99`) avec `worldSeed=coinage-mvp-seed`.

## Résultat clé
- Le système génère **de vraies familles visuelles distinctes** (pas seulement de faibles variations), mais la diversité est partiellement **compressée** par des clamps/guards et par le LOD galaxy.

## Chiffres factuels (échantillon runtime = 100)
- Archétypes distincts observés: 12/12.
- Catégories de surface observées: 9/9.
- Profils de terrain observés: 5/6 (pas de `moderate` observé car `pickTerrainProfile` retourne directement par archétype).
- Palettes observées: 14/14.
- Taille écran (Galaxy View, 1080p): médiane ≈ 92.5 px de diamètre, min ≈ 67.3 px, max ≈ 147.2 px.
- Clustering (greedy, features visuelles + couleurs):
  - seuil 0.12 → 100 familles / 100 planètes
  - seuil 0.18 → 95 familles / 100 planètes
  - seuil 0.24 → 80 familles / 100 planètes

## Points d’écrasement identifiés
- `oceanLevel` clampé à `[0.05, 0.62]` + garde-fous `minLandRatio`, `maxOceanRatio`, `mountainLevel - 0.18`.
- `mountainLevel` très resserré (std faible).
- LOD galaxy réduit résolution et relief micro:
  - `meshResolution` plafonné à 14/18
  - `ridgedStrength *= 0.9`
  - `elevationCap *= 0.95`
  - `detailAttenuation *= 0.84`
- Éclairage galaxy global uniforme (ambient + 2 directional) et tone mapping fixe.

## Conclusion binaire
- Verdict: **A) de vrais types de planètes distincts**.
- Nombre pratique de familles visuelles perceptibles (Galaxy View): **~80 à 95 sur 100** selon seuil de similarité visuelle appliqué.
- Limitation principale: **combinaison LOD galaxy + clamps hydrologie/relief + échelle écran** (pas un effondrement au niveau du générateur seul).
